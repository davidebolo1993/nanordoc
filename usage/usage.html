# Usage

A typical MinION/GridION sequencing run produces the output tree below.

sample_dir/
|-- fast5_fail
|   |-- sample_0.fast5
|   |-- sample_1.fast5
|   `-- ...
|-- fast5_pass
|   |-- sample_0.fast5
|   |-- sample_1.fast5
|   `-- ...
|-- fast5_skip
|   |-- sample_0.fast5
|   |-- sample_1.fast5
|   `-- ...
|-- fastq_fail
|   |-- sample_0.fastq
|   |-- sample_1.fastq
|   `-- ...
|-- fastq_pass
|   |-- sample_0.fastq
|   |-- sample_1.fastq
|   `-- ...
`-- sample_sequencing_summary.txt

NanoR assumes that the required inputs follow the aforementioned output tree.

# Data analysis and visualization

NanoR allows to analyze/visualize sequencing data from MinION/GridION starting both from basecalled FAST5 or a combination of sequencing summary and FASTQ.

# Basecalled FAST5

A typical workflow to analyze/visualize basecalled FAST5 is described below.

library(NanoR)

sequencing_run<-"/path/to/sample_dir"
pass<-file.path(sequencing_run,"fast5_pass")
fail<-file.path(sequencing_run,"fast5_fail")
skip<-file.path(sequencing_run,"fast5_skip")
output<-file.path("/path/to/result_dir")

Mlist<-NanoPrepareM(DataPass=pass,DataFail=fail,DataSkip=skip, MultiRead=TRUE) #multi-read FAST5 are the default output format. Simply switch to FALSE if dealing with single-read FAST5
Mtable<-NanoTableM(NanoMList=Mlist, DataOut=output, Cores=6, GCC=FALSE) #increase the number of cores to increase speed if dealing with a large number of FAST5; switch GCC to TRUE to enable calculation of GC content
NanoStatsM(NanoMList=Mlist, NanoMTable=Mtable, DataOut=output, KeepGGObj = FALSE) #to store also table behind ggplot2-plots, switch KeepGGObj to TRUE

This will generate a metadata table (that is, a table containing key informations extracted from FAST5), a summary containing the major statistics and several plots that offer a comprehensive overview of the sequencing run. Have a look at the NanoR open-access paper for some of the plots generated.

TIP

Older versions of NanoR included also a function to extract FASTQ from basecalled FAST5 (NanoFastqM). This function has been dismissed, as there's no such a need with the current outputs.

# Sequencing summary and FASTQ

A typical workflow to analyze/visualize sequencing summary and FASTQ is described below. This is a lot faster than the approach described above.

library(NanoR)

sequencing_run<-"/path/to/sample_dir"
summary<-file.path(sequencing_run,"sample_sequencing_summary.txt")
fastq<-file.path(sequencing_run, "fastq_pass")
output<-file.path("/path/to/result_dir")

Glist<-NanoPrepareG(DataSummary=summary,DataFastq=fastq)
Gtable<-NanoTableG(NanoGList=Glist, DataOut=output, GCC=FALSE) # switch GCC to TRUE to enable calculation of GC content from passed FASTQ.
NanoStatsG(NanoGList=Glist, NanoGTable=Gtable, DataOut=output, KeepGGObj = FALSE) #to store also table behind ggplot2-plots, switch KeepGGObj to TRUE

TIP

Older versions of NanoR included also a function to filter FASTQ (NanoFastqG). This function has been dismissed as there are faster ways to do this outside the R environemnt.

# Experiments comparison

NanoR allows users to compare their analyzed sequencing runs in one command, exploiting the "DataForComparison" folders generated during the analysis step.

library(NanoR)

inputs<-c("/path/to/output1/DataForComparison","/path/to/output2/DataForComparison","/path/to/output3/DataForComparison")
labels<-c("S1","S2","S3")
output<-"/path/to/comparison_output"
NanoCompare(DataIn=inputs,DataOut=output,Labels=labels)
